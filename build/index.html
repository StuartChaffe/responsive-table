Jazcss

Jazcss is my frontend framework that I use whenever I start a new project. It contains all the usual things you'd expect from any framework, plus a few extras. It's built to be responsive, customisable and semantically correct as much as possible. This site is using the latest version of Jazcss and shows off all the things it can do.

It's worth noting that you really shouldn't consider using this framework unless you can be sure gzip will be enabled on the server. This is because Jazcss offers a shit ton of optional spacing classes to make fine-tuning layouts easier. They're implemented using iterative SCSS @each blocks. This pushes the size of the framework way up to around 200KB, but because it's all very similar code, gzip compression brings it down by about 95% making it really lightweight afterwards.

The heavy stuff is split into its own CSS file, core.min.css. Huge, right? Bung it in an online GZIP compressor like this one, hit the CSS button and see the magic.

Tech

  • Gulp - JS task runner. This is the master file of Jazcss and is what brings everything together to serve up a built site
  • Handlebars - HTML templating engine. Makes it easy to reuse components on multiple pages or create page layouts
  • Browsersync - Live browser reloading, CSS injection, static site serving, session synchronisation
  • Sass - CSS precompiler providing advanced CSS functionality. Jazcss uses SCSS instead of Sass itself

Directory structure

  • src/pages - The main view files which get rendered into html files with the same name when built
  • src/layouts - Wrappers for pages. Contains placeholders which page files are rendered inside of
  • src/partials - Individual component markup files. These can be used on pages and layouts using {{>header}}
  • src/styles - Sass styles, described below
  • src/scripts - JS scripts. Files in these directories are automatically concatenated and minified via gulp
  • src/images, src/videos, src/fonts - These directories are directly copied into the build folder
  • .editorconfig - See EditorConfig.org. Standardises formatting for project so tabs don't fight with spaces

Style structure

  • Base - Contains core framework files and shouldn't need much changing other than variables.scss
  • Helpers - Classes that provide useful functionality. Shouldn't need changing
  • Layout - Classes specifically focussed on element positioning. Shouldn't need changing
  • Common - Style rules that target standard HTML elements such as <a>, <h1> etc
  • Components - The bread and butter of your project. Every partial should have a corresponding file here
  • Vendor - Shove all your dirty third-party CSS in here
