Buttons vs Link-buttons

The difference between a button and a link-button is important, as each has specific behaviour that should belong to it. A <button>, should always have a type attribute attached. If you have an element that should trigger a client-side action, use type="button. For a form submit, use type="submit". This means you should no longer be using the <input type="button"/> or <input type="submit"/> elements.

A link-button is a link that's styled to look the same as a <button> element. To use one, simply add the btn class to a <a> tag. Only use a link-button if there is a href to attach and no JS event hooked up.

<button type="button">Button</button>

Link-button <a href="" class="btn">Link-button</a>

One of the key reasons for using the <button> element, is that it can contain anything. Images, divs, even tables are all valid inside buttons. This framework uses font-awesome for simple icons, which you should use with buttons like so:

<button> <i class="fa fa-search" aria-hidden="true"></i><span>Search</span> </button>

Of course, you can do the same thing with link-buttons:

Search Search <a href="" class="btn"> <i class="fa fa-search" aria-hidden="true"></i><span>Search</span> </a>
